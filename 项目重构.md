#### 3.4 资源生成并行化 + 失败降级

**改动**：`lib/learning-graph/nodes/generate-resources.ts`

```typescript
// 并行生成 + 失败降级
const results = await Promise.allSettled(
  resourcePlan.execution.resourceTypes.map(async (type) => {
    write({ type: 'agent_status', agentId: type, status: 'running', resourceType: type });
    try {
      const generated = await generateResource(type, node.knowledgePoints, state.profile, state.aiConfig);
      const resource: Resource = { id: crypto.randomUUID(), userId: state.userId ?? 'current', type, ... };
      write({ type: 'agent_status', agentId: type, status: 'completed', resourceType: type });
      return resource;
    } catch (error) {
      write({ type: 'agent_status', agentId: type, status: 'failed', resourceType: type });
      return { id: crypto.randomUUID(), type, status: 'failed', content: String(error), ... } as Resource;
    }
  })
);
// Promise.allSettled 完成后统一推送 resource_delta，保证事件顺序可控
const generatedResources = results.filter(r => r.status === 'fulfilled').map(r => (r as PromiseFulfilledResult<Resource>).value);
for (const resource of generatedResources) write({ type: 'resource_delta', resource });
```

**同时修复**：
- edges 应累积所有历史边，而非只记录最后一条
- catch 块必须返回 `generatedResources`（已成功生成的资源不能丢弃）

**write() 并发安全**：Node.js 单线程无真正并发问题，但异步交错导致事件顺序不确定。采用"先收集后统一推送"方案，牺牲实时性换取可靠性。通过 `agent_status.running` 事件在开始时推送弥补——前端展示"各 Agent 工作中"，完成后一次性展示结果。

#### 3.5 修复评估闭环

**改动**：`plan-resources.ts`、`plan-node.ts`

`LearningState` 已有 `evaluationResult` 字段，evaluate 节点写入，update_profile 读取 `profileUpdate` 子字段。但 `plan-node`/`plan-resources` 完全不读取。

**方案**：
1. **`plan-resources.ts`**：当 `state.evaluationResult?.weakPoints` 存在时，将 `allowLLM` 设为 `true`，利用 `app/api/resource-decision/route.ts` 已有的 LLM 决策层（L166-210）动态决定资源类型调整策略，**不硬编码** boostTypes/suppressTypes
2. **`plan-node.ts`**：在 `buildPrompt` 中注入 `evaluationResult.weakPoints`，优先规划薄弱知识点节点
3. **`evaluationScore`**：前端评估页面展示 + 作为 `allowLLM` 门控条件（低评分时启用 LLM 增强）

#### 3.6 修复类型系统不一致

- `Interests.preferredFormats` 补充 `reading` | `ppt`
- `ConversationMessage.timestamp` 统一为 ISO string
- `Quiz.answer` 区分单选（`string`）和多选（`string[]`），或使用判别联合
- `RESOURCE_TYPE_LABELS` 统一为一份定义，删除 `resource-generators.ts` 中的重复
- `LearningPathNode.prerequisites` 改为节点 ID 引用（`string[]`），而非描述文本
