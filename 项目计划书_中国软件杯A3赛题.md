## 1. 赛题核心要求速览

### 1.1 必做功能（占分 45%+）

| 功能         | 硬性指标                   | 对应 OpenMAIC 能力       |
| ---------- | ---------------------- | -------------------- |
| 对话式学习画像构建  | ≥6 维度，支持随学随新           | 多智能体对话 + 结构化输出解析     |
| 多智能体协同资源生成 | ≥5 种资源类型，不同角色 Agent 协作 | LangGraph 编排 + 生成管道  |
| 个性化学习路径规划  | 动态路径 + 精准推送            | Director Graph 状态机模式 |

### 1.2 加分功能

| 功能     | 价值              |
| ------ | --------------- |
| 智能辅导   | 多模态答疑（文字+图解+视频） |
| 学习效果评估 | 多维度评估 + 动态调整    |

### 1.3 非功能性要求

- 界面美观、流式输出、Markdown 渲染、多模态内容卡片化
- 鼓励使用开源前端
- AI 代码须标注来源
- 响应时间合理，避免长时间空白等待
- 建议优先使用科大讯飞工具

### 4.2 Agent 详细设计

#### Profile Agent（画像构建 Agent）

| 属性           | 描述                                                                                         |
| ------------ | ------------------------------------------------------------------------------------------ |
| **职责**       | 通过多轮对话抽取学生特征，构建和动态更新学习画像                                                                   |
| **输入**       | 用户对话消息 + 当前画像数据                                                                            |
| **输出**       | 结构化画像更新（JSON）+ 对话回复                                                                        |
| **工具**       | `extract_profile_features`（结构化特征抽取）、`update_profile`（画像更新）、`query_profile_history`（画像历史查询） |
| **画像维度（≥6）** | 1.知识基础 2.认知风格 3.学习目标 4.薄弱知识点 5.学习时间偏好 6.兴趣方向 7.学习节奏 8.易错点偏好                                |

**画像 Schema 设计**：

```typescript
interface LearningProfile {
```

### 4.3 Agent 协作工作流

#### 工作流 1：新用户画像构建

```
用户进入 → Profile Agent 发起对话
  → 多轮对话抽取特征（3-5 轮）
  → 结构化输出画像 JSON
  → 存储画像到数据库
  → 触发 Path Agent 规划初始学习路径
  → 触发资源生成工作流
```

#### 工作流 2：资源生成（核心工作流）

```
用户请求/路径规划触发 → Orchestrator 接收
  → 读取学生画像
  → 并行调度：
     ├─ Document Agent → 生成讲解文档
     ├─ Quiz Agent → 生成练习题
     ├─ MultiModal Agent → 生成思维导图
     └─ Code Agent → 生成代码案例（如适用）
  → 等待所有 Agent 完成
  → Path Agent 整合资源到学习路径
  → SSE 流式推送结果到前端
```

#### 工作流 3：学习效果评估（加分项）

```
用户完成学习/练习 → Evaluation Agent 分析
  → 收集学习行为数据
  → 分析练习成绩和错误模式
  → 更新画像（薄弱点、易错点）
  → 触发 Path Agent 调整路径
  → 触发资源 Agent 补充针对性资源
```

### 4.4 LangGraph 状态机设计

```typescript
import { StateGraph, END } from '@langchain/langgraph';

interface AgentState {
  userId: string;
  profile: LearningProfile;
  currentTask: 'profile_build' | 'resource_gen' | 'path_plan' | 'tutor' | 'evaluate';
  messages: BaseMessage[];
  generatedResources: Resource[];
  learningPath: LearningPath | null;
  activeAgents: string[];
  errors: string[];
}

const workflow = new StateGraph<AgentState>({
  channels: {
    userId: { value: null },
    profile: { value: null },
    currentTask: { value: null },
    messages: { value: (x: BaseMessage[], y: BaseMessage[]) => x.concat(y) },
    generatedResources: { value: (x: Resource[], y: Resource[]) => x.concat(y) },
    learningPath: { value: null },
    activeAgents: { value: null },
    errors: { value: (x: string[], y: string[]) => x.concat(y) },
  },
});

// 添加节点
workflow.addNode('orchestrator', orchestratorNode);
workflow.addNode('profile_agent', profileAgentNode);
workflow.addNode('document_agent', documentAgentNode);
workflow.addNode('quiz_agent', quizAgentNode);
workflow.addNode('multimodal_agent', multimodalAgentNode);
workflow.addNode('code_agent', codeAgentNode);
workflow.addNode('path_agent', pathAgentNode);
workflow.addNode('tutor_agent', tutorAgentNode);

// 添加边
workflow.addEdge('__start__', 'orchestrator');
workflow.addConditionalEdges('orchestrator', routeToAgent, {
  profile: 'profile_agent',
  resource_gen: 'document_agent',  // 并行触发
  path_plan: 'path_agent',
  tutor: 'tutor_agent',
});
workflow.addEdge('profile_agent', 'path_agent');
workflow.addEdge('document_agent', 'path_agent');
workflow.addEdge('quiz_agent', 'path_agent');
workflow.addEdge('multimodal_agent', 'path_agent');
workflow.addEdge('code_agent', 'path_agent');
workflow.addEdge('path_agent', END);
workflow.addEdge('tutor_agent', END);
```

### 6.1 对话式学习画像构建

#### 用户流程

```
1. 用户首次进入 → 显示欢迎引导
2. Profile Agent 发起对话："你好！我是你的学习助手，想了解一下你的学习情况..."
3. 多轮对话（3-8 轮），自然抽取特征：
   - "你目前学过哪些编程语言？" → 知识基础
   - "你更喜欢看视频还是读文档？" → 认知风格
   - "你学习的主要目标是什么？" → 学习目标
   - "哪些知识点你觉得比较困难？" → 薄弱点
   - "你每天大概能学多长时间？" → 时间偏好
   - "你对哪些方向比较感兴趣？" → 兴趣方向
4. 实时展示画像雷达图（6+ 维度）
5. 用户确认/修改画像
6. 画像保存，触发路径规划
```

#### 技术实现要点

- 使用 `callLLM` 流式生成对话回复
- 每轮对话后，用结构化输出解析画像特征更新
- 前端使用 `StreamBuffer` 实现打字机效果
- 画像变更历史可追溯（版本化）
- 支持用户主动修改画像维度

### 6.3 个性化学习路径规划

#### 用户流程

```
1. 画像构建完成 → 自动触发 Path Agent
2. Path Agent 分析画像 + 学习目标
3. 生成学习路径（知识点序列 + 资源推荐 + 时间安排）
4. 前端展示路径时间线/知识图谱
5. 用户按路径学习 → 完成节点 → 解锁下一节点
6. 练习/评估结果 → 动态调整路径
```

#### 路径规划算法

```
输入：学生画像 + 学习目标 + 知识图谱
处理：
  1. 确定目标知识点集合
  2. 根据知识基础确定起点
  3. 拓扑排序确定学习顺序（考虑前置依赖）
  4. 根据薄弱点调整权重（薄弱知识点分配更多时间和资源）
  5. 根据时间偏好安排学习节奏
  6. 为每个节点匹配合适资源
输出：学习路径（有序节点列表 + 资源推荐 + 时间估算）
```

#### 关键 API

```
POST /api/path/plan
  Body: { userId, goal }
  Response: SSE 流
    event: agent_start  data: { agent: "path" }
    event: text_delta   data: { text: "正在分析你的学习需求..." }
    event: path_update  data: { path: LearningPath }
    event: agent_end    data: { agent: "path" }

GET /api/path/:userId
  Response: { path: LearningPath }

POST /api/path/:pathId/progress
  Body: { nodeId, status, quizResult? }
  Response: { path: LearningPath, adjustments? }
```

```typescript
// 讯飞星火 API 调用示例（WebSocket 流式）
const SPARK_API_URL = 'wss://spark-api.xf-yun.com/v1/chat';

// 认证方式：APIKey + APISecret 生成鉴权 URL
function generateAuthUrl(apiKey: string, apiSecret: string): string {
  // HMAC-SHA256 签名
  const timestamp = new Date().toUTCString();
  const signatureOrigin = `host: spark-api.xf-yun.com\ndate: ${timestamp}\nGET /v1/chat HTTP/1.1`;
  const signature = crypto.createHmac('sha256', apiSecret).update(signatureOrigin).digest('base64');
  const authorization = btoa(`api_key="${apiKey}", algorithm="hmac-sha256", headers="host date request-line", signature="${signature}"`);
  return `${SPARK_API_URL}?authorization=${authorization}&date=${encodeURIComponent(timestamp)}&host=spark-api.xf-yun.com`;
}

// Vercel AI SDK 适配器
import { createSparkModel } from '@/lib/ai/spark-adapter';
const sparkModel = createSparkModel('spark-4.0-turbo', { apiKey, apiSecret });
```
