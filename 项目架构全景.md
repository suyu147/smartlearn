# SmartLearn 项目架构全景文档

> 基于 Next.js 15 + LangGraph 的多智能体个性化学习系统
> 赛题：第十五届中国软件杯大学生软件设计大赛 A3 赛题

---

## 一、项目概览

SmartLearn 是一个基于大模型的个性化资源生成与学习多智能体系统。核心流程：

**对话构建画像 → 规划学习路径 → 多 Agent 协同生成资源 → 学习评估 → 反馈调整策略（闭环）**

| 维度 | 技术 |
|:---|:---|
| 前端框架 | Next.js 15 (App Router, Turbopack) + React 19 |
| 语言 | TypeScript 5.7 |
| 样式 | Tailwind CSS 4 + shadcn/ui + Radix UI |
| 状态管理 | Zustand 5 + persist (localStorage) + immer |
| AI 编排 | LangGraph 0.3 + LangChain Core 0.3 + Vercel AI SDK 5 |
| 数据库 | Prisma 6 + PostgreSQL（schema 已定义，未接入业务层） |
| 图表/可视化 | recharts, echarts, @xyflow/react, markmap |
| 动画 | motion (framer-motion) |
| i18n | i18next + react-i18next（4 语言：zh-CN/en-US/ja-JP/ru-RU） |
| AI 提供商 | 12 家（OpenAI/Anthropic/Google/GLM/Qwen/DeepSeek/Kimi/MiniMax/SiliconFlow/Doubao/Grok/Spark） |

## 二、核心架构 — 两套 LangGraph 图

项目存在**两套独立的 LangGraph 图**，职责严格分离，不可混淆：

### 2.1 learning-graph（学习流程图）

- **入口 API**：`app/api/learn/route.ts`（POST，SSE 流式）
- **图定义**：`lib/learning-graph/graph.ts`
- **模块入口**：`lib/learning-graph/index.ts`

**流程拓扑**：

```
START → routeByAction →
  ├─ 'start'         → plan_node → analyze_learner → plan_resources → generate_resources → END
  ├─ 'node_complete'  → evaluate → update_profile → plan_node（闭环继续）
  ├─ 'quiz_result'    → evaluate → update_profile → END
  └─ 'tutor_chat'     → tutor_respond → END
```

**闭环机制**：当 `action === 'node_complete'` 时，`evaluate → update_profile → plan_node` 形成闭环，持续学习。

### 2.2 director-graph（PPT 演示图）

- **入口 API**：`app/api/chat/route.ts`（POST，SSE 流式）
- **图定义**：`lib/orchestration/director-graph.ts`

**流程拓扑**：

```
START → director →
  ├─ end → END
  └─ next → agent_generate → director（循环）
```

**职责**：PPT/Stage 多 Agent 对话编排，支持白板动作、Agent 调度、流式输出。

## 三、learning-graph 节点详解

### 3.1 路由节点

| 文件 | 导出 | 功能 |
|:---|:---|:---|
| `lib/learning-graph/nodes/router.ts` | `routeByAction` | 根据 `state.action` 分发：`start`→plan_node，`node_complete`/`quiz_result`→evaluate，`tutor_chat`→tutor_respond |

### 3.2 学习流程节点

| 文件 | 导出 | 功能 |
|:---|:---|:---|
| `lib/learning-graph/nodes/plan-node.ts` | `planNode` | 调用 LLM 根据画像+目标+已完成节点，规划下一个学习节点（标题、2-4 知识点、预估时长） |
| `lib/learning-graph/nodes/analyze-learner.ts` | `analyzeLearnerNode` | 纯同步分析，提取学习者快照（偏好格式、薄弱主题、测验分数、学习阶段），不调用 LLM |
| `lib/learning-graph/nodes/plan-resources.ts` | `planResourcesNode` | 调用 `decideNodeResourcePlan` 决策资源类型，支持用户手动覆盖 |
| `lib/learning-graph/nodes/generate-resources.ts` | `generateResourcesNode` | 逐类型调用 `generateResource` 生成资源，PPT 额外调用 `generatePptScenes` |
| `lib/learning-graph/nodes/evaluate.ts` | `evaluateNode` | 评估测验结果，调用 LLM 生成薄弱点/强项/建议/画像更新 |
| `lib/learning-graph/nodes/update-profile.ts` | `updateProfileNode` | 将评估的画像更新应用到学习者画像 |
| `lib/learning-graph/nodes/tutor-respond.ts` | `tutorRespondNode` | AI 导师对话，流式生成回复 |

### 3.3 Helper 函数

| 文件 | 导出 | 功能 |
|:---|:---|:---|
| `lib/learning-graph/helpers/evaluate.ts` | `evaluateQuizResults` | 测验评估核心逻辑：计算分数 + LLM 评估 |
| `lib/learning-graph/helpers/resource-generators.ts` | `generateResource` | 资源生成统一入口：document 两步生成，video 搜索聚合，其余 LLM 生成 |
| `lib/learning-graph/helpers/tutor.ts` | `streamTutorResponse` | 导师回复核心逻辑，附加资源上下文 |
| `lib/learning-graph/helpers/ppt-generator.ts` | `generatePptScenes` | PPT 场景生成：大纲→配图→场景构建 |

### 3.4 类型定义

| 文件 | 关键导出 |
|:---|:---|
| `lib/learning-graph/types.ts` | `QuizResultPayload`, `EvaluationResultPayload`, `LearnRequest`, `LearnerSnapshot`, `LearnEvent`（联合类型） |
| `lib/learning-graph/state.ts` | `LearningState`, `LearningStateType` |

## 四、LearningState 状态结构

定义于 `lib/learning-graph/state.ts`，使用 LangGraph `Annotation.Root`：

| 字段 | 类型 | 说明 |
|:---|:---|:---|
| `action` | `LearnRequest['action']` | 当前动作（start/evaluate/tutor 等） |
| `sessionId` | `string` | 会话 ID |
| `profile` | `ProfileDimensions` | 学习者画像（8 维度） |
| `goal` | `string` | 学习目标 |
| `completedNodes` | `LearningPathNode[]` | 已完成节点 |
| `currentNode` | `LearningPathNode \| null` | 当前学习节点 |
| `currentNodeId` | `string \| null` | 当前节点 ID |
| `quizResults` | `QuizResultPayload[]` | 答题结果 |
| `generatedResources` | `Resource[]` | 已生成资源（reducer 累加） |
| `evaluationResult` | `EvaluationResultPayload \| null` | 评估结果（**已写入但下游未消费**） |
| `evaluationScore` | `number \| null` | 评估分数（**已写入但下游未消费**） |
| `resourcePlan` | `ResourceDecisionResultV2 \| null` | 资源决策结果 |
| `learnerSnapshot` | `LearnerSnapshot \| null` | 学习者快照 |
| `pptScenes` | `Scene[] \| null` | PPT 场景 |
| `aiConfig` | `LearnRequest['aiConfig']` | AI 配置（provider/model/key） |
| `resourceFeedback` | `PriorNodeFeedback[]` | 历史资源反馈 |
| `nodeDecisionOverrides` | `Record<string, ResourceType[]>` | 用户手动覆盖的资源类型 |
| `conversationHistory` | `Array<{role, content}>` | 对话历史 |
| `attachedResources` | `Array<{id, type, title, content}>` | 附加资源 |
| `updatedProfile` | `ProfileDimensions \| null` | 更新后的画像 |
| `phase` | `string` | 当前阶段 |

## 十一、数据库 Schema

定义于 `prisma/schema.prisma`，8 个模型：

```
User ─┬─ LearningProfile (1:N)
      ├─ Resource (1:N)
      ├─ LearningPath (1:N)
      ├─ ChatSession (1:N)
      └─ QuizResult (1:N)

PathNodeResource (M:N 关联表)
StageOutline (独立)
```

**枚举**：
- `ResourceType`：document / mindmap / quiz / video / code / reading（**缺少 ppt**）
- `ResourceStatus`：generating / ready / failed
- `PathStatus`：active / completed / archived
- `SessionType`：profile_build / tutor / resource_request

## 十六、核心数据流

### 16.1 学习流程数据流

```
用户输入学习目标
       │
       ▼
  ┌─────────┐    SSE 流     ┌──────────────┐
  │ 前端     │ ◄────────── │ /api/learn    │
  │ Workspace│              │  (POST)       │
  └────┬─────┘              └──────┬───────┘
       │                           │
       │  POST {action:'start',    │  compileLearningGraph()
       │  profile, goal, aiConfig} │  .invoke(state, {writer})
       │                           │
       ▼                           ▼
  ┌─────────────────────────────────────────┐
  │           learning-graph                 │
  │                                          │
  │  plan_node ──► analyze_learner           │
  │       │              │                   │
  │       │              ▼                   │
  │       │     plan_resources               │
  │       │              │                   │
  │       │              ▼                   │
  │       │     generate_resources ──► END   │
  │       │                                  │
  │  (闭环: evaluate → update_profile →      │
  │         plan_node → ...)                 │
  └──────────────────────────────────────────┘
       │
       │  SSE 事件流:
       │  node_ready → resource_delta → ppt_delta → done
       │
       ▼
  前端 Zustand Store 更新:
  useLearningPathStore ← node_ready/path_update
  useResourcesStore    ← resource_delta
  useStageStore        ← ppt_delta
  useLearningProfileStore ← profile_update
```

### 16.2 画像构建数据流

```
用户对话
    │
    ▼
┌──────────┐   POST    ┌──────────────────┐
│ ProfileChat│ ───────► │ /api/profile/chat │
└──────────┘           └────────┬─────────┘
                                │
                                │  LLM 回复 + 提取画像维度
                                │
                                ▼
                    ┌──────────────────────┐
                    │ SSE 事件流:           │
                    │  text_delta (回复文本) │
                    │  profile_update (维度) │
                    └──────────┬───────────┘
                               │
                               ▼
                    useLearningProfileStore.updateDimensions()
```

### 16.3 PPT 生成数据流

```
用户输入主题
    │
    ▼
┌──────────┐   POST    ┌──────────────────┐
│  PPT Page │ ───────► │ /api/generate/ppt │
└──────────┘           └────────┬─────────┘
                                │
                                │  三阶段流水线:
                                │  1. generateSceneOutlinesFromRequirements
                                │  2. batchGenerateImages (可选)
                                │  3. buildSceneFromOutline × N
                                │
                                ▼
                    ┌──────────────────────┐
                    │ SSE 事件流:           │
                    │  outline_delta        │
                    │  image_delta          │
                    │  scene_delta          │
                    │  generation_complete  │
                    └──────────┬───────────┘
                               │
                               ▼
                    useStageStore.addScene()
```
