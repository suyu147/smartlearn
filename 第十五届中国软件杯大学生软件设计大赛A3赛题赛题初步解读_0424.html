赛题名称

基于大模型的个性化资源生成与学习多智能体系统开发

赛题简介

在数字化与智能化深度融合的时代，高等教育的个性化变革成为核心发展方向，同时也面临传统教学模式适配性不足的挑战。不同学生在知识基础、学习能力、兴趣方向上的显著差异，使得标准化教学难以满足个性化学习需求，部分学生存在知识吸收效率低的问题。

技术机遇

当前大模型技术迎来高速发展新阶段，以通用大模型、多模态生成大模型（如SeeDance等）、AI辅助编程工具（如Claude Code等）为代表的技术体系，具备强大的自然语言理解、多模态内容生成、代码辅助开发及实时推理能力，为高等教育领域的创新升级带来全新契机。

本赛题旨在借助大模型技术体系，融合前沿AI技术，突破传统教育的技术与模式局限，要求参赛团队构建高等教育个性化学习资源体系，开发智能学习智能体系统，切实满足学生的个性化、多模态学习需求。

赛题业务场景

在高等教育学习过程中，学生普遍面临学习资源繁杂无序、难以精准匹配自身需求，且缺乏智能化、个性化学习指导的核心问题。

核心痛点
  • 不同专业、不同学历水平的学生面对海量课程资料、学术文献、学习辅助工具时，难以快速筛选出契合自身学习进度和能力的资源
  • 课堂集体讲授模式无法兼顾每位学生的学习节奏与特点，导致学生在知识掌握和能力提升上存在明显差距
  • 传统学习模式及基础的智能辅助系统，因缺乏多模态生成、多智能体协同等前沿AI技术的支撑，难以满足现代高等教育对培养创新型、个性化人才的要求

本赛题要求参赛团队构建多智能体系统，为学生打造专属的个性化资源学习智能体，并借助多智能体协作实现智能化、精准化的学习引导。系统需依托各类高等教育资源，融合多模态生成、代码辅助开发等技术，以某一具体专业课程（如人工智能、计算机、电子信息相关等）为切入点，实现个性化资源的自动化生成与建设。

因材施教 → 数字化落地 → 真正实现"因材施教"的数字化落地

基本功能需求

参赛团队需深入调研和研究新时代大学生的学习需求和痛点，融合前沿AI技术和工具，开发出能够高效生成个性化、多模态学习资源的智能体系统。

核心目标

实现提升学生学习效率、优化学习资源利用、增强学习效果。系统应包含以下核心功能：

对话式学习画像自主构建

摒弃传统繁琐表单，支持通过自然语言对话（结合学生的专业、学习目标、学习历史等）自动抽取特征，构建包含不少于6个维度（如知识基础、认知风格、易错点偏好等）的动态学生画像，并支持画像的随学随新。

多智能体协同的资源生成

系统需体现"多智能体"架构设计，通过与学生的智能交互，大模型结合AI前沿技术和工具，依据学生提供的专业、课程内容、知识短板、学习需求等信息，生成针对性的多模态学习资料。

需由不同角色的Agent协作完成至少5种类型的个性化资源生成：
  • 专业课程讲解文档
  • 知识点思维导图
  • 不同类型练习题目
  • 拓展阅读材料
  • 多模态教学视频/动画
  • 代码类实操案例等

个性化学习路径规划和资源推送

依托多智能体协同工作机制，整合系统生成的个性化资源，结合大模型对学生专业、学习进度、知识掌握情况及学习偏好的深度分析，为学生规划科学、动态的个性化学习路径，明确学习步骤和顺序；同时基于画像实现学习资源的精准推送，涵盖文档、视频、题库、实操案例等多类型内容。

智能辅导（可选加分项）

当学生在学习过程中遇到问题时，系统提供即时、多模态的答疑解惑服务，通过智能体的数据分析、大模型的知识支持，结合多模态生成技术，为学生提供详细的文字解答、图解说明、短视频讲解等多样化解答形式，实现针对性学习引导。

学习效果评估（可选加分项）

通过实时跟踪学生的学习行为、练习测试情况、资源使用反馈等数据，依托大模型的数据分析能力实现对学生学习效果的多维度、精准评估；并根据评估结果及时动态调整学习资源推送策略和学习计划，实现学习方案的持续优化。

非功能性需求

  • 系统界面美观大方、简洁明了，交互逻辑清晰，符合现代AI产品交互规范（如流式输出、Markdown渲染、多模态内容卡片化展示），无明显功能与交互缺陷，可接受AI辅助实现界面设计与交互优化
  • 鼓励优先使用开源项目前端
  • AI代码/框架：须在提交的代码中明确位置标注名称、来源及其协作要求
  • 系统需具备基本的、可用的、具有数据安全的人机交互机制，确保生成的学生画像真实、资源内容无违规信息
  • 各功能模块的功能调用及响应时间在合理范围内，满足学生的日常使用场景；多模态资源生成的响应效果满足实际学习场景需求，并提供可追踪、渐进式呈现、流式等机制，避免长时间空白等待

  • 在开发过程中，建议采用模块化架构设计，保持代码结构清晰
  • 建议优先使用科大讯飞相关工具

赛题技术深度解读

本赛题要求构建一个基于大模型的多智能体协同系统，核心是通过Agent架构实现个性化学习资源的自动化生成与智能推送。这不是简单的”AI聊天机器人”，而是需要多个专业Agent协作完成复杂任务的智能体系统。

赛题本质

这是一个AI Native教育应用的完整实现——从学生画像构建、多模态资源生成、学习路径规划到智能辅导的全流程，每个环节都需要大模型和多智能体技术的深度参与。

核心技术挑战分析

挑战一：多智能体架构设计

赛题明确要求”多智能体”架构，这意味着不能只有一个通用Agent，而是需要设计多个专业化Agent协同工作。每个Agent有明确的职责分工，通过消息传递或共享状态协作完成任务。

常见误区

  • 只有一个大模型接口，没有Agent分工
  • Agent之间没有协作机制
  • 所有功能都塞进一个Prompt
  • 缺乏Agent状态管理和记忆

推荐做法

  • 至少设计5个专业Agent（画像、文档、题库、视频、路径规划等）
  • 使用成熟的多智能体框架（LangGraph/AutoGen）
  • 设计Agent间的消息协议和工作流
  • 为每个Agent配置专属工具和记忆

挑战二：多模态资源生成

赛题要求生成至少5种类型的学习资源，包括文档、思维导图、题目、视频/动画等。这需要整合多种生成模型和工具，并确保生成内容的质量和一致性。

硬性要求

必须生成：专业课程讲解文档、知识点思维导图、不同类型练习题目、拓展阅读材料、多模态教学视频/动画、代码类实操案例（至少5种）

挑战三：学习画像的动态构建

要求通过对话自动抽取特征构建≥6个维度的学生画像，并支持”随学随新”。这需要设计画像Schema、特征抽取策略和增量更新机制。

挑战四：个性化与精准推送

系统需要根据学生画像动态调整资源生成策略和学习路径，实现真正的”因材施教”。这要求画像数据能有效指导Agent的决策过程。

系统架构建议

多智能体架构设计

1

画像构建Agent

职责：通过对话抽取学生特征（专业、年级、知识基础、学习风格、目标、薄弱点等≥6维度），构建和更新学生画像

工具：讯飞星火API + 结构化输出解析 + 数据库存储

2

文档生成Agent

职责：根据知识点和学生画像生成个性化讲解文档（Markdown格式，包含概念、原理、案例、总结）

工具：讯飞星火 + Prompt模板 + Markdown渲染

3

题库生成Agent

职责：生成不同难度和类型的练习题（选择题、填空题、编程题、案例分析题等）

工具：讯飞星火 + 题目Schema + 自动评分逻辑

4

多模态资源Agent

职责：生成思维导图、教学视频/动画、配图等多模态内容

工具：Markmap（思维导图）+ SeeDance（视频）+ Stable Diffusion（配图）

5

代码实操Agent

职责：生成可运行的代码示例和实操案例（带注释和运行说明）

工具：Claude Code / GitHub Copilot + 代码执行沙箱

6

路径规划Agent

职责：根据学生画像和学习目标，规划个性化学习路径（知识点顺序、资源推荐、时间安排）

工具：讯飞星火X1.5（推理模型）+ 知识图谱 + 推荐算法

7

智能辅导Agent（可选加分）

职责：实时答疑解惑，提供多模态解答（文字+图解+视频）

工具：讯飞星火 + RAG检索 + 多模态生成

对话交互 → 画像构建 → 多Agent协同生成资源 → 路径规划 → 精准推送 → 智能辅导

技术架构分层

接入层：Web前端 + 移动端（可选）+ 对话界面（支持流式输出）

编排层：LangGraph/AutoGen多智能体编排引擎 + 工作流管理

能力层：讯飞星火API + SeeDance + 向量检索 + 代码生成工具

数据层：PostgreSQL（结构化数据）+ Milvus（向量数据）+ Redis（缓存）+ OSS（多媒体文件）

得分策略与建议

评分权重：创新价值与实用性（35%）+ 功能实现及技术要求（45%）+ 配套文档（10%）+ 演示视频/PPT（10%）= 100%

创新价值与实用性（35分）—— 差异化竞争

建议创新点
  • Agent协作可视化：实时展示多个Agent的协作过程（如”文档Agent正在生成...题库Agent等待中...”），让评委看到多智能体架构的实际运作
  • 学习画像可解释性：不只是存储画像数据，还要可视化展示”为什么推荐这个资源”（基于画像的哪些维度）
  • 资源质量评估：设计自动化的资源质量评分机制（准确性、难度适配度、完整性等）
  • 跨模态一致性：确保同一知识点的文档、视频、题目在内容上保持一致，形成完整的学习闭环

功能实现及技术要求（45分）—— 硬指标必达

  • 画像维度≥6：专业、年级、知识基础、学习风格、目标、薄弱点、学习时间偏好、认知风格等
  • 资源类型≥5：文档、思维导图、题目、视频/动画、代码案例（必须全部实现）
  • 多智能体架构：至少5个专业Agent，有明确的协作机制和工作流
  • 流式输出：大模型生成内容要支持流式展示，避免长时间等待
  • 多模态展示：前端要支持Markdown渲染、思维导图交互、视频播放、代码高亮等

配套文档（10分）—— 体现专业性

  • 系统架构图（清晰展示多智能体架构和数据流）
  • 技术选型说明（为什么选择这些工具和模型）
  • 多智能体协作机制设计文档
  • 部署说明书（Docker一键部署）
  • API文档（如果提供API接口）

演示视频/PPT（10分）—— 讲好故事

演示重点
  • 开场：用一个真实学习场景引入（如”小明是计算机专业大二学生，想学习机器学习但不知从何开始...”）
  • 核心功能演示：完整展示从对话构建画像 → 多Agent生成资源 → 路径规划 → 资源推送的全流程
  • 技术亮点：重点展示多智能体协作过程、多模态资源生成效果、个性化推荐逻辑
  • 效果对比：展示”传统学习方式 vs 本系统”的对比，突出价值

实用建议

  • 选择一个具体专业课程深入做（如”机器学习”或”数据结构”），比泛泛覆盖多课程效果更好
  • 准备真实测试数据（至少20个学生画像样本，100+学习资源）
  • 提前申请讯飞开放平台API额度，避免临近比赛时额度不足
  • 前端界面参考成熟的在线教育产品（如Coursera、Khan Academy），注重用户体验
  • 代码规范、注释完整，便于评委审阅
