> 版本：v2.0 | 日期：2026-06-06 | 基于用户最新需求重写

***

## 1. 产品故事线

```
用户进入 SmartLearn
  → 【阶段1】与 AI 对话，构建个人学习画像（8维度）
  → 画像构建完成后 → 自动跳转到【学习工作台】
  → 【阶段2】AI 基于画像自动生成学习路径（5-8个节点），显示在左侧
  → 【阶段3】系统自动检测已解锁节点，提前调用资源生成API，批量生成全部资源（文档/题目/思维导图/视频/代码/拓展阅读/PPT）
  → 用户点击左侧节点 → 中间区域显示该节点的所有资源（含动态PPT）
  → 【阶段4】按节点逐步学习，完成后解锁下一节点
  → 【阶段5】每完成一个节点（含测验），自动更新学习画像，形成闭环
```

## 3. 使用流程详解

### 3.1 阶段1：画像构建（复用现有一切，不修改）

**路由**：`/profile`（`app/profile/page.tsx`）

**流程**：与现有完全一样，用户与 AI 多轮对话构建8维画像。

**触发跳转**：当画像完整性达标后（≥4个维度有非默认值），弹窗提示"画像构建完成！即将进入学习工作台"，2秒后自动跳转到 `/workspace`。

```
画像完整性判断 → 弹窗 → 自动跳转 /workspace
```

**画像完整性判断**：复用 `lib/utils/profile-utils.ts`（文档v1中已有设计，≥4个维度非默认值即为完成）。

### 3.2 阶段2：路径自动生成（进入工作台后立即执行）

**触发时机**：用户进入 `/workspace` 页面，检测到有画像但无路径 → 自动调用路径规划 API。

**调用 API**：

```
POST /api/path/plan
{
  goal: "从画像中提取的学习目标",
  profile: profile.dimensions,
  aiConfig: { providerId, modelId, apiKey, baseUrl }
}
```

**数据流**：

```
进入 /workspace
  → useEffect 检测：profile 存在 && path === null
  → 自动从画像 shortTerm/longTerm 拼接 goal
  → fetch POST /api/path/plan
  → SSE 解析 path_update 事件
  → useLearningPathStore.setPath(path)
  → 左侧面板渲染路径节点
  → 自动触发阶段3：资源预生成
```

**生成的路径结构**：

```typescript
// 路径包含 5-8 个节点，第一个节点状态为 in_progress，其余为 locked
{
  id: "uuid",
  goal: "学习目标",
  nodes: [
    { id: "n1", title: "节点1", status: "in_progress", knowledgePoints: [...], resources: [] },
    { id: "n2", title: "节点2", status: "locked",    knowledgePoints: [...], resources: [] },
    // ... 5-8 nodes
  ],
  edges: [...],
  estimatedDays: 14,
}
```

### 3.3 阶段3：资源预生成（路径生成后自动批量生成）

**核心规则**：

- 路径生成后，自动扫描所有 `status !== 'locked'`（即 `available` 或 `in_progress`）且 `resources` 为空或资源不足的节点
- 对每个这样的节点，调用资源生成 API 为该节点的 `knowledgePoints` 生成全部资源
- **提前生成**，用户点击节点时资源已就绪，零等待

**调用 API**（逐个节点调用）：

对每个需要生成资源的节点：

```
POST /api/generate/resources
{
  knowledgePoints: node.knowledgePoints,
  resourceTypes: ["document", "mindmap", "quiz", "video", "code", "reading"],
  profile: profile.dimensions,
  aiConfig: { providerId, modelId, apiKey, baseUrl }
}
```

```
POST /api/generate/ppt
{
  requirement: "讲解：" + node.knowledgePoints.join("、"),
  language: "zh-CN",
  aiConfig: { providerId, modelId, apiKey, baseUrl }
}
```

**生成策略**：

```
资源预生成调度器（前端实现，串行避免API过载）：

for each node in (in_progress + available nodes):
  if node 资源为空:
    → 设置 node 为 "资源生成中..." 状态
    → 并行调用 /api/generate/resources 和 /api/generate/ppt
    → SSE 接收 resource_delta，注入 node.resources 引用
    → SSE 接收 ppt scene_ready，创建 Resource(type='ppt')
    → 生成完成，更新左侧节点显示资源数量
```

**左边节点显示效果**：

```
┌──────────────────────┐
│ ▶ 节点1：Python基础语法  │
│   📄文档 🧠导图 📝题目    │ ← 点击任意资源图标即可在中间查看
│   💻代码 🎬视频 🎯PPT    │
├──────────────────────┤
│ 🔒 节点2：面向对象编程   │ ← 灰色未解锁
│   生成中... 3/6        │
├──────────────────────┤
│ 🔒 节点3：文件操作      │ ← 灰色未解锁
└──────────────────────┘
```

### 3.4 阶段4：节点学习（点击左侧节点 → 中间显示资源）

**用户操作流程**：

```
1. 用户点击左侧 "节点1：Python基础语法" 下的 📄文档
2. 中间区域全屏展示该文档（DocumentViewer）
3. 用户阅读完成后，可：
   a. 点击 📝题目 → 中间显示 QuizPlayer 做练习
   b. 点击 🎯PPT → 中间显示内嵌PPT播放或跳转全屏
   c. 点击 🧠导图 → 中间显示 MindmapViewer
   d. 点击右侧聊天框，与 Tutor 讨论疑惑
4. 全部资源学习完毕后，点击 "节点完成" → 解锁下一节点
```

**中间区域资源渲染器**（根据 ResourceType 分发）：

```tsx
// 中间区域核心渲染逻辑
function ResourceViewer({ resource }: { resource: Resource }) {
  switch (resource.type) {
    case 'document': return <DocumentViewer content={resource.content} title={resource.title} />;
    case 'mindmap':  return <MindmapViewer content={resource.content} title={resource.title} />;
    case 'quiz':     return <QuizPlayer content={resource.content} title={resource.title} />;
    case 'code':     return <CodeRunner content={resource.content} title={resource.title} />;
    case 'video':    return <VideoPlayer content={resource.content} title={resource.title} videoData={resource.metadata?.videoData} />;
    case 'reading':  return <DocumentViewer content={resource.content} title={resource.title} />;
    case 'ppt':      return <PPTViewer scenes={resource.metadata?.pptData} />;
    default:         return <DocumentViewer content={resource.content} title={resource.title} />;
  }
}
```

### 3.5 阶段5：闭环反馈（每完成一个节点更新画像）

**触发时机**：

1. 用户点击"节点完成"按钮
2. 用户完成该节点下的测验

**更新流程**：

```
节点完成
  → 调用 POST /api/evaluate（结合 quizResults + profile）
  → SSE 返回 evaluation_result（含 profileUpdate）
  → 前端 updateDimensions(profileUpdate)
  → 画像实时更新
  → 解锁下一节点
  → 对新解锁节点触发资源预生成（阶段3）
```

**路径解锁规则**：

```
节点1 [in_progress] → 点击完成 → 节点1 [completed]
  → 检测：节点2 的前置条件（prerequisites）已满足
  → 节点2 [locked] → [available]
  → 触发节点2 的资源预生成
```

### 5.1 画像完整性判断工具

**文件**：`lib/utils/profile-utils.ts`（新文件）

```typescript
import type { ProfileDimensions } from '@/lib/types/profile';

export function isProfileComplete(dimensions: ProfileDimensions | null): boolean {
  if (!dimensions) return false;

  let filledCount = 0;

  if (dimensions.knowledgeBase?.subjects?.length > 0) filledCount++;
  if (dimensions.cognitiveStyle?.type && dimensions.cognitiveStyle.type !== 'reading') filledCount++;
  if (dimensions.learningGoals?.shortTerm?.length > 0) filledCount++;
  if (dimensions.weakPoints?.topics?.length > 0) filledCount++;
  if (dimensions.timePreference?.preferredDuration > 0) filledCount++;
  if (dimensions.interests?.domains?.length > 0) filledCount++;
  if (dimensions.learningPace?.speed && dimensions.learningPace.speed !== 'moderate') filledCount++;
  if (dimensions.errorPatterns?.commonMistakes?.length > 0) filledCount++;

  return filledCount >= 4;
}

export function calculateProfileCompleteness(dimensions: ProfileDimensions | null): number {
  if (!dimensions) return 0;
  const totalDimensions = 8;
  let filledCount = 0;

  if (dimensions.knowledgeBase?.subjects?.length > 0) filledCount++;
  if (dimensions.cognitiveStyle?.type) filledCount++;
  if (dimensions.learningGoals?.shortTerm?.length > 0 || dimensions.learningGoals?.longTerm) filledCount++;
  if (dimensions.weakPoints?.topics?.length > 0 || dimensions.weakPoints?.errorPatterns?.length > 0) filledCount++;
  if (dimensions.timePreference?.preferredDuration > 0) filledCount++;
  if (dimensions.interests?.domains?.length > 0) filledCount++;
  if (dimensions.learningPace?.speed) filledCount++;
  if (dimensions.errorPatterns?.commonMistakes?.length > 0 || dimensions.errorPatterns?.difficultAreas?.length > 0) filledCount++;

  return Math.round((filledCount / totalDimensions) * 100);
}
```

### 5.3 新增学习工作台页面

**文件**：`app/workspace/page.tsx`（新文件）

```tsx
'use client';

import { useEffect, useState, useCallback } from 'react';
import { useRouter } from 'next/navigation';
import { useLearningPathStore } from '@/lib/store/learning-path';
import { useLearningProfileStore } from '@/lib/store/learning-profile';
import { useResourcesStore } from '@/lib/store/resources';
import { useSettingsStore } from '@/lib/store/settings';
import { isProfileComplete } from '@/lib/utils/profile-utils';
import { LearningPathPanel } from '@/components/workspace/learning-path-panel';
import { ResourceViewer } from '@/components/workspace/resource-viewer';
import { TutorChatPanel } from '@/components/workspace/tutor-chat-panel';
import { WorkspaceHeader } from '@/components/workspace/workspace-header';
import type { Resource, ResourceType } from '@/lib/types/resource';

export default function WorkspacePage() {
  const router = useRouter();
  const { profile } = useLearningProfileStore();
  const { path, setPath, updateNodeStatus } = useLearningPathStore();
  const { resources, addResource } = useResourcesStore();
  const { providerId, modelId, apiKey, baseUrl } = useSettingsStore();

  const [selectedResource, setSelectedResource] = useState<Resource | null>(null);
  const [generatingNodes, setGeneratingNodes] = useState<Set<string>>(new Set());
  const [isPlanning, setIsPlanning] = useState(false);

  // 1. 未完成画像 → 跳回 /profile
  useEffect(() => {
    if (!isProfileComplete(profile?.dimensions ?? null)) {
      router.replace('/profile');
    }
  }, [profile]);

  // 2. 有画像无路径 → 自动规划
  useEffect(() => {
    if (profile && !path && !isPlanning) {
      autoPlanPath();
    }
  }, [profile, path]);

  async function autoPlanPath() {
    setIsPlanning(true);
    try {
      const dimensions = profile!.dimensions;
      const goal = dimensions.learningGoals.shortTerm?.join('；')
        || dimensions.learningGoals.longTerm
        || dimensions.knowledgeBase.subjects.map(s => s.name).join('、');

      const response = await fetch('/api/path/plan', {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({
          goal,
          profile: dimensions,
          aiConfig: { providerId, modelId, apiKey, baseUrl },
        }),
      });

      const reader = response.body?.getReader();
      const decoder = new TextDecoder();
      let buffer = '';

      while (reader) {
        const { done, value } = await reader.read();
        if (done) break;
        buffer += decoder.decode(value, { stream: true });
        const lines = buffer.split('\n');
        buffer = lines.pop() || '';
        for (const line of lines) {
          if (line.startsWith('data: ')) {
            try {
              const data = JSON.parse(line.slice(6));
              if (data.type === 'path_update' && data.path) {
                // 初始话全部节点为 in_progress(首个) 和 locked(其余)
                setPath(data.path);
              }
            } catch { /* skip */ }
          }
        }
      }
    } catch (error) {
      console.error('Path planning error:', error);
    } finally {
      setIsPlanning(false);
    }
  }

  // 3. 路径生成后 → 预生成已解锁节点的所有资源
  useEffect(() => {
    if (path) {
      preGenerateAllUnlockedResources();
    }
  }, [path?.id]);

  async function preGenerateAllUnlockedResources() {
    if (!path) return;
    const unlockedNodes = path.nodes.filter(
      n => n.status !== 'locked' && n.resources.length === 0
    );
    for (const node of unlockedNodes) {
      setGeneratingNodes(prev => new Set(prev).add(node.id));
      try {
        // 并行：资源 + PPT
        const types: ResourceType[] = ['document', 'mindmap', 'quiz', 'video', 'code', 'reading'];
        
        const [resourcesRes, pptRes] = await Promise.all([
          fetch('/api/generate/resources', {
            method: 'POST',
            headers: { 'Content-Type': 'application/json' },
            body: JSON.stringify({
              knowledgePoints: node.knowledgePoints,
              resourceTypes: types,
              profile: profile?.dimensions ?? null,
              aiConfig: { providerId, modelId, apiKey, baseUrl },
            }),
          }),
          fetch('/api/generate/ppt', {
            method: 'POST',
            headers: { 'Content-Type': 'application/json' },
            body: JSON.stringify({
              requirement: '讲解：' + node.knowledgePoints.join('、'),
              language: 'zh-CN',
              aiConfig: { providerId, modelId, apiKey, baseUrl },
            }),
          }),
        ]);

        // 解析资源 SSE
        await parseResourceSSE(resourcesRes, node.id);
        
        // 解析 PPT SSE
        await parsePPTSSE(pptRes, node.id);
      } catch (error) {
        console.error(`Failed to generate resources for node ${node.id}:`, error);
      } finally {
        setGeneratingNodes(prev => {
          const next = new Set(prev);
          next.delete(node.id);
          return next;
        });
      }
    }
  }

  async function parseResourceSSE(response: Response, nodeId: string) {
    const reader = response.body?.getReader();
    const decoder = new TextDecoder();
    let buffer = '';
    while (reader) {
      const { done, value } = await reader.read();
      if (done) break;
      buffer += decoder.decode(value, { stream: true });
      const lines = buffer.split('\n');
      buffer = lines.pop() || '';
      for (const line of lines) {
        if (line.startsWith('data: ')) {
          try {
            const data = JSON.parse(line.slice(6));
            if (data.type === 'resource_delta' && data.resource) {
              addResource(data.resource);
              // 关联资源到节点
              updateNodeResource(nodeId, data.resource);
            }
          } catch { /* skip */ }
        }
      }
    }
  }

  async function parsePPTSSE(response: Response, nodeId: string) {
    const reader = response.body?.getReader();
    const decoder = new TextDecoder();
    let buffer = '';
    let scenes: any[] = [];
    while (reader) {
      const { done, value } = await reader.read();
      if (done) break;
      buffer += decoder.decode(value, { stream: true });
      const lines = buffer.split('\n');
      buffer = lines.pop() || '';
      for (const line of lines) {
        if (line.startsWith('data: ')) {
          try {
            const data = JSON.parse(line.slice(6));
            if (data.type === 'generation_complete' && data.scenes) {
              const pptResource: Resource = {
                id: crypto.randomUUID(),
                userId: 'current',
                type: 'ppt',
                title: `${path?.nodes.find(n => n.id === nodeId)?.title} - 动态课件`,
                content: JSON.stringify(data.scenes),
                sourceAgent: 'ppt',
                status: 'ready',
                createdAt: new Date().toISOString(),
                metadata: { pptData: data.scenes },
              };
              addResource(pptResource);
              updateNodeResource(nodeId, pptResource);
            }
          } catch { /* skip */ }
        }
      }
    }
  }

  function updateNodeResource(nodeId: string, resource: Resource) {
    // 使用 learning-path store 的现有节点更新逻辑
    // 将 resource 引用关联到节点
    const node = path?.nodes.find(n => n.id === nodeId);
    if (node) {
      const updatedNodes = path!.nodes.map(n => {
        if (n.id === nodeId) {
          return {
            ...n,
            resources: [...n.resources, {
              resourceId: resource.id,
              type: resource.type,
              title: resource.title,
            }],
          };
        }
        return n;
      });
      setPath({ ...path!, nodes: updatedNodes });
    }
  }

  // 4. 节点完成 → 解锁下一节点 + 更新画像 + 预生成新节点资源
  async function handleNodeComplete(nodeId: string) {
    updateNodeStatus(nodeId, 'completed');

    // 解锁下一个节点
    if (path) {
      const currentIndex = path.nodes.findIndex(n => n.id === nodeId);
      const nextNode = path.nodes[currentIndex + 1];
      if (nextNode) {
        updateNodeStatus(nextNode.id, 'available');
        // 触发新节点资源预生成
        setTimeout(() => preGenerateAllUnlockedResources(), 100);
      }
    }

    // 调用评估更新画像
    // ... 调用 POST /api/evaluate
  }

  return (
    <div className="flex h-screen flex-col">
      <WorkspaceHeader profile={profile} />
      <div className="flex flex-1 overflow-hidden">
        <LearningPathPanel
          path={path}
          generatingNodes={generatingNodes}
          onSelectResource={setSelectedResource}
          onNodeComplete={handleNodeComplete}
        />
        <div className="flex-1 overflow-auto border-x">
          <ResourceViewer resource={selectedResource} />
        </div>
        <TutorChatPanel />
      </div>
    </div>
  );
}
```

```tsx
'use client';

import { useState, useRef, useEffect } from 'react';
import { Button } from '@/components/ui/button';

import { Send, ChevronLeft, ChevronRight, MessageSquare, Loader2 } from 'lucide-react';
import { useSettingsStore } from '@/lib/store/settings';

interface ChatMessage {
  id: string;
  role: 'user' | 'assistant';
  content: string;
}

export function TutorChatPanel() {
  const [collapsed, setCollapsed] = useState(false);
  const [messages, setMessages] = useState<ChatMessage[]>([]);
  const [input, setInput] = useState('');
  const [isLoading, setIsLoading] = useState(false);
  const scrollRef = useRef<HTMLDivElement>(null);
  const { providerId, modelId, apiKey, baseUrl } = useSettingsStore();

  useEffect(() => {

    }

          <ChevronLeft className="h-4 w-4" />
        </Button>
      </div>
    );
  }

  async function handleSend() {
    if (!input.trim() || isLoading) return;
    const userMsg: ChatMessage = { id: crypto.randomUUID(), role: 'user', content: input };
    setMessages(prev => [...prev, userMsg]);
    setInput('');
    setIsLoading(true);

    try {
      const response = await fetch('/api/tutor/chat', {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({
          message: input,
          conversationHistory: messages.slice(-10).map(m => ({ role: m.role, content: m.content })),
          aiConfig: { providerId, modelId, apiKey, baseUrl },
        }),
      });

      const reader = response.body?.getReader();
      const decoder = new TextDecoder();
      const assistantId = crypto.randomUUID();
      let assistantContent = '';

      setMessages(prev => [...prev, { id: assistantId, role: 'assistant', content: '' }]);

      let buffer = '';
      while (reader) {
        const { done, value } = await reader.read();
        if (done) break;
        buffer += decoder.decode(value, { stream: true });
        const lines = buffer.split('\n');
        buffer = lines.pop() || '';
        for (const line of lines) {
          if (line.startsWith('data: ')) {
            try {
              const data = JSON.parse(line.slice(6));
              if (data.type === 'text_delta') {
                assistantContent += data.text;
                setMessages(prev => prev.map(m =>
                  m.id === assistantId ? { ...m, content: assistantContent } : m
                ));
              }
            } catch { /* skip */ }
          }
        }
      }
    } catch (error) {
      console.error('Tutor chat error:', error);
    } finally {
      setIsLoading(false);
    }
  }

  return (
    <div className="w-[320px] border-l flex flex-col min-w-0">
      <div className="flex items-center justify-between p-3 border-b">
        <h3 className="font-medium text-sm flex items-center gap-2">
          <MessageSquare className="h-4 w-4" />
          智能辅导
        </h3>
        <Button variant="ghost" size="icon" onClick={() => setCollapsed(true)} title="折叠">
          <ChevronRight className="h-4 w-4" />
        </Button>
      </div>

      <ScrollArea className="flex-1 p-3" ref={scrollRef}>
        {messages.length === 0 && (
          <div className="text-center text-sm text-muted-foreground py-8">
            <MessageSquare className="mx-auto h-8 w-8 mb-2 opacity-30" />
            学习过程中有任何疑问，随时向 AI 助教提问
          </div>
        )}
        <div className="space-y-3">
          {messages.map(msg => (
            <div key={msg.id} className={`flex ${msg.role === 'user' ? 'justify-end' : 'justify-start'}`}>
              <div className={`max-w-[85%] rounded-lg px-3 py-2 text-sm ${
                msg.role === 'user'
                  ? 'bg-primary text-primary-foreground'
                  : 'bg-muted'
              }`}>
                {msg.content || (msg.role === 'assistant' && isLoading ? (
                  <Loader2 className="h-3 w-3 animate-spin" />
                ) : msg.content)}
              </div>
            </div>
          ))}
        </div>
      </ScrollArea>

      <div className="p-3 border-t">
        <div className="flex gap-2">
          <Textarea
            value={input}
            onChange={e => setInput(e.target.value)}

                handleSend();
              }
            }}

            className="min-h-[40px] resize-none text-sm"

            disabled={isLoading}
          />

            {isLoading ? <Loader2 className="h-4 w-4 animate-spin" /> : <Send className="h-4 w-4" />}
          </Button>
        </div>
      </div>
    </div>
  );
```

## 6. 资源预生成调度器（前端实现）

**核心逻辑**：路径生成后，自动对已解锁节点触发批量资源生成。

```
资源预生成流程：

1. path 生成成功（path_update 事件）
   ↓
2. 扫描 path.nodes，收集 status !== 'locked' 的节点
   ↓
3. 对每个节点，并行调用两个 API：
   - POST /api/generate/resources  (生成文档/思维导图/题目/视频/代码/阅读)
   - POST /api/generate/ppt         (生成动态课件)
   ↓
4. SSE 实时解析：
   - resource_delta → addResource() → 关联到节点 resources 列表
   - generation_complete → ppt 资源创建
   ↓
5. 左侧面板显示节点资源数量，生成中显示加载动画
   ↓
6. 全部完成后，用户点击节点即可零等待查看资源
```

**生成状态管理**：

- `generatingNodes: Set<string>` — 正在生成的节点ID集合
- 生成中节点在左侧显示 Loading 动画
- 生成完成后移除 loading 状态

## 10. 关键设计决策

### 10.1 为什么不新增后端 API？

现有6个 API（`/api/profile/chat`、`/api/generate/resources`、`/api/generate/ppt`、`/api/path/plan`、`/api/tutor/chat`、`/api/evaluate`）已覆盖全部需求。新前端直接调用现有 API，零后端改动，零风险。

### 10.2 为什么提前预生成所有资源？

用户体验：用户点击左侧节点时，资源已就绪，零等待。而非点击后等待生成时间（可能数十秒）。

代价：资源生成期间有一定API调用开销，但资源可缓存（存入 Zustand Store + localStorage）。

### 10.3 为什么保留独立 PPT 页面？

内嵌PPT播放器适合快速浏览，但全屏PPT播放（含键盘控制、动画效果）需要独立页面。两个入口互补。

### 10.4 为什么左侧缩小 PathTimeline 而不是重写？

`PathTimeline` 和 `PathProgress` 已验证可用，功能完整。通过 CSS `scale(0.85)` + `origin-top-left` 缩放嵌入左侧面板，零重复开发。

### 10.5 为什么删除旧页面？

`/resources`、`/tutor`、`/learning-path` 三个独立页面的功能全部合并到 `/workspace` 三栏布局中。保留旧页面会造成功能重复和维护负担。
